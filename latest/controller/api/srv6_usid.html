controller.srv6_usid.encode_endpoint_node(node, grpc_ip, grpc_port, fwd_engine, locator, udt=None)[source]¶

Get a dict-representation of a node (endpoint of the path), starting from gRPC IP and port, uDT sid, forwarding engine and locator.

Parameters

  • node (str) – Node identifier. This could be a name, a SID (IPv6 address) or a number (uSID identifier).

  • grpc_ip (str) – gRPC IP address of the node.

  • grpc_port (int) – Port number of the gRPC server.

  • udt (str, optional) – uDT SID of the node, used for the decap operation. If not provided, the uDT SID is not added to the SID list.

  • fwd_engine (str) – Forwarding engine to be used (e.g. Linux or VPP).

  • locator (str) – Locator part of the SIDs (e.g. fcbb:bbbb::).

Returns

Dict representation of the node. The dict has the following fields: - name - grpc_ip - grpc_port - uN - uDT - fwd_engine

Return type

dict

Raises

InvalidConfigurationError – If the node params are invalid.

controller.srv6_usid.encode_intermediate_node(node, locator)[source]¶

Get a dict-representation of a node (intermediate node of the path), starting from gRPC IP and port, uDT sid, forwarding engine and locator. For the intermediate nodes, we don’t need uDT, forwarding engine. gRPC IP and gRPC address.

Parameters

  • node (str) – Node identifier. This could be a name, a SID (IPv6 address) or a number (uSID identifier).

  • locator (str) – Locator part of the SIDs (e.g. fcbb:bbbb::).

Returns

Dict representation of the node. The dict has the following fields: - name - grpc_ip (set to None) - grpc_port (set to None) - uN - uDT (set to None) - fwd_engine (set to None)

Return type

dict

Raises

InvalidConfigurationError – If the node params are invalid.

controller.srv6_usid.fill_nodes_info(nodes_info, nodes, l_grpc_ip=None, l_grpc_port=None, l_fwd_engine=None, r_grpc_ip=None, r_grpc_port=None, r_fwd_engine=None, decap_sid=None, locator=None)[source]¶

Fill ‘nodes_info’ dict with the nodes containined in the ‘nodes’ list.

Parameters

  • nodes_info (dict) – Dict containined the nodes information where to add the nodes.

  • nodes (list) – List of nodes. Each node can be expressed as SID (IPv6 address), a uSID identifier (integer) or a name.

  • l_grpc_ip (str, optional) – gRPC address of the left node in the path.

  • l_grpc_port (str, optional) – Port number of the gRPC server on the left node of the path.

  • l_fwd_engine (str, optional) – Forwarding engine to be used on the left node of the path (e.g. Linux or VPP).

  • r_grpc_ip (str, optional) – gRPC address of the right node in the path.

  • r_grpc_port (str, optional) – Port number of the gRPC server on the right node of the path.

  • r_fwd_engine (str, optional) – Forwarding engine to be used on the right node of the path (e.g. Linux or VPP).

  • decap_sid (str, optional) – Decap SID. This could be a SID (IPv6 address) or a uSID identifier (an integer).

  • locator (str, optional) – Locator part of the SIDs (e.g. fcbb:bbbb::).

Raises

InvalidConfigurationError – If the node params are invalid.

controller.srv6_usid.handle_srv6_usid_policy(operation, lr_destination=None, rl_destination=None, nodes_lr=None, nodes_rl=None, table=-1, metric=-1, persistency=True, _id=None, l_grpc_ip=None, l_grpc_port=None, l_fwd_engine=None, r_grpc_ip=None, r_grpc_port=None, r_fwd_engine=None, decap_sid=None, locator=None, db_conn=None)[source]¶

Handle a SRv6 Policy using uSIDs

Parameters

  • operation (str) – The operation to be performed on the uSID policy (i.e. add, get, change, del)

  • destination (str) – Destination of the SRv6 route

  • nodes (list) – Waypoints of the SRv6 route

  • device (str, optional) – Device of the SRv6 route. If not provided, the device is selected automatically by the node.

  • table (int, optional) – Routing table containing the SRv6 route. If not provided, the main table (i.e. table 254) will be used.

  • metric (int, optional) – Metric for the SRv6 route. If not provided, the default metric will be used.

  • l_grpc_ip (str, optional) – gRPC IP address of the left node, required if the left node is expressed numerically in the nodes list.

  • l_grpc_port (str, optional) – gRPC port of the left node, required if the left node is expressed numerically in the nodes list.

  • l_fwd_engine (str, optional) – forwarding engine of the left node, required if the left node is expressed numerically in the nodes list.

  • r_grpc_ip (str, optional) – gRPC IP address of the right node, required if the right node is expressed numerically in the nodes list.

  • r_grpc_port (str, optional) – gRPC port of the right node, required if the right node is expressed numerically in the nodes list.

  • r_fwd_engine (str, optional) – Forwarding engine of the right node, required if the right node is expressed numerically in the nodes list.

  • decap_sid (str, optional) – uSID used for the decap behavior (End.DT6).

  • locator (str, optional) – Locator prefix (e.g. ‘fcbb:bbbb::’).

Returns

Status Code of the operation (e.g. 0 for STATUS_SUCCESS)

Return type

int

Raises

  • NodeNotFoundError – Node name not found in the mapping file

  • InvalidConfigurationError – The mapping file is not a valid YAML file

  • TooManySegmentsError – segments arg contains more than 6 segments

  • SIDLocatorError – SID Locator is wrong for one or more segments

  • InvalidSIDError – SID is wrong for one or more segments
