controller.srv6_usid.handle_srv6_usid_policy(operation, nodes_filename=None, lr_destination=None, rl_destination=None, nodes_lr=None, nodes_rl=None, table=-1, metric=-1, persistency=True, _id=None)[source]¶

Handle a SRv6 Policy using uSIDs

Parameters

  • operation (str) – The operation to be performed on the uSID policy (i.e. add, get, change, del)

  • nodes_filename (str) – Name of the YAML file containing the mapping of node names to IP addresses

  • destination (str) – Destination of the SRv6 route

  • nodes (list) – Waypoints of the SRv6 route

  • device (str, optional) – Device of the SRv6 route. If not provided, the device is selected automatically by the node.

  • table (int, optional) – Routing table containing the SRv6 route. If not provided, the main table (i.e. table 254) will be used.

  • metric (int, optional) – Metric for the SRv6 route. If not provided, the default metric will be used.

Returns

Status Code of the operation (e.g. 0 for STATUS_SUCCESS)

Return type

int

Raises

  • NodeNotFoundError – Node name not found in the mapping file

  • InvalidConfigurationError – The mapping file is not a valid YAML file

  • TooManySegmentsError – segments arg contains more than 6 segments

  • SIDLocatorError – SID Locator is wrong for one or more segments

  • InvalidSIDError – SID is wrong for one or more segments
